01What was actually run

The root config.json declares the intended design. What's on disk, read per-run, doesn't fully match it — both in scope and in a contamination pattern seen before in this project's other results directories.

  1. Declared design. benchmark_config: f ∈ {0,2,3,4,5}, γ (data_distribution_parameter) ∈ {1.0, 0.33}, 4 aggregators (GeometricMedian, CenteredClipping, TrMean, MultiKrum) × NNM+ARC pre-aggregation, 3 attacks (Optimal_ALittleIsEnough_neg1, SignFlipping, Optimal_InnerProductManipulation), 5 training seeds (42–46), 500 steps, evaluation every 50 steps (11 checkpoints per run). model.encoding.time_steps ∈ {10,25,50}, model_params.beta ∈ {0.95,0.99}, alpha=[1.2] fixed, threshold=1.0 fixed.
  2. Actual grid is asymmetric, not a full cross-product. Of the theoretical 720 (T×β×agg×attack×f×γ) combinations, only 360 exist on disk, split cleanly by γ: γ=1.0 only has f=0 (72 combos — a clean, unattacked baseline grid over T×β); γ=0.33 only has f∈{2,3,4,5} (288 combos, f=0 never run at this γ). Design gap The two axes that matter most for the collapse story — heterogeneity and attack strength — are never varied independently of each other. There is no run that isolates "non-IID but unattacked" or "IID but attacked."
  3. Alpha contamination in one corner. 10 of the 72 γ=1.0/f=0 combos are actually alpha=0.5 rather than the declared 1.2 — all at T=10, β=0.95. This is a leftover from an earlier/different config that wrote into the same results_directory, the same pattern found earlier in robust_new_atan_sweep for the threshold sweep's baseline search.
  4. Seed completeness: 359/360 combos have all 5 seeds. The one exception — T=50, β=0.99, CenteredClipping, ALIE, f=4, γ=0.33 — has only 4 (seed 46 missing).
  5. Checkpoint completeness: 1,797 of 1,849 rows have the full 11 evaluation points. 50 rows have 9–10 instead of 11, and every one of them sits in the same corner as the alpha contamination (T=10, β=0.95, γ=1.0, f=0). They still reach sane final accuracy (~91–92%), so these read as leftover partial-eval logs from an earlier attempt at that corner, not failed training runs — the same overwrite pattern as fact 3, not a new problem.
  6. 2 genuinely broken runs. T=50, β=0.99, GeometricMedian, seed 42 and T=50, β=0.99, MultiKrum, seed 45 (both at f=2, γ=0.33) have only 1 evaluation point (the step-0 initialization) and are frozen at 0.1822 accuracy — training was killed almost immediately. Same corner as fact 4 Both crashed runs and the one missing seed all live in T=50, β=0.99 — the single heaviest configuration in the grid (longest unroll, least-leaky membrane). Consistent with resource pressure (OOM or disk) hitting the most expensive cells first, the same class of issue as the disk-full crash on the threshold sweep.
  7. No firing-rate or per-client-vector instrumentation present. This sweep predates that instrumentation entirely — only the older per-step gradient-geometry metrics exist (honest_var_trace, honest_mean_cos_sim, honest_grad_norm_*, sparsity metrics, per-client loss/accuracy). No honest_firing_rate, no layer_firing_rate, no client_vectors.

04Interpretation

Everything above is what's in the files. What follows is my reading of it — treat it as a hypothesis to check, not a conclusion.

This looks like it contradicts the standing null result on T/β

CLAUDE.md's null-result list states: "sweeping α/T/β barely moved the SNN's gradient geometry or its collapse boundary." For ALIE against T, that's not what this data shows. At T=10 the network survives ALIE gracefully out to f=3–4; the instant T becomes 25 or 50, it is at chance from f=2 onward, for every aggregator. That's a boundary moving from "graceful to f≈3" to "dead at the first tested f" — about as large a movement as this kind of sweep can register. Worth reconciling: either the earlier null-result claim was based on a different attack, a narrower T range, or the effect wasn't visible without gamma=0.33 specifically.

The effect is attack-specific, not a generic "T changes gain" story

If longer T simply shrank the honest gradient's effective gain (per the gain-collapse hypothesis), all three attacks should degrade similarly as T grows. They don't: ALIE flips from survivable to catastrophic; IPM flips the other way (T=25/50 are more robust than T=10 at f=2); Sign Flipping is roughly T-invariant except for the β=0.99 interaction at T=50. That asymmetry suggests the mechanism is about how each attack's constructed vector interacts with the temporal-averaging geometry specifically, not a single scalar "gain" shared across attacks. ALIE calibrates its perturbation to the honest-client variance; if longer T changes that variance's shape in a way ALIE's calibration exploits precisely while IPM's inner-product-maximization doesn't, that would produce exactly this pattern. The gradient-geometry files already sitting in these run directories (honest_var_trace, honest_mean_cos_sim, honest_normalized_*) could test this directly without any new runs.

The β=0.99 / T=50 / Sign-Flipping interaction fits the dead-attractor story, but only in one corner

A near-integrator membrane (β→1, minimal leak) combined with a long integration window (T=50) is exactly the condition CLAUDE.md's positive-feedback hypothesis predicts should be most fragile — and Sign Flipping, the "dumbest" attack, is the one that catches it here. But it's oddly specific: the same β/T combination doesn't visibly worsen ALIE (already dead either way) or IPM (only a partial drop). If this mechanism is real and general, I'd expect it to show up as a monotonic trend across β at fixed T=50 for all three attacks, not an isolated cell.

What this sweep can't tell us yet

Because γ=1.0 was only run at f=0 and γ=0.33 only at f≥2, there's no way to tell from this data alone whether the ALIE/T cliff is a property of the attack itself or requires non-IID data to trigger. A small, targeted follow-up — T∈{10,25}, β=0.95, ALIE only, γ=1.0, f∈{2,3,4,5}, 4 aggregators, 5 seeds — would be about 160 runs and would directly close that gap.

Operational note

The 2 crashed runs and the missing seed all cluster in the single most expensive cell (T=50, β=0.99). Same failure signature as the disk-full crash on the threshold sweep. Worth watching disk/memory headroom specifically when T=50 configs are in the queue.
